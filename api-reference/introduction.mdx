title: 'Pikzels API'
description: 'REST API for image generation and manipulation services'
---

## Base URL

All API requests should be made to:

```
https://api.pikzels.com
```

## Authentication

All API endpoints require authentication using an API key. Include your API key in the request headers:

```bash
X-Api-Key: YOUR_API_KEY
```

Your API key should be a valid Pikzels API key starting with `pkz_`.

<Warning>
  Keep your API key secure and never share it publicly. Do not commit it to version control.
</Warning>

## Rate Limits

- Maximum of 10 concurrent requests per API key
- Requests exceeding this limit will receive a 429 error

## Response Format

All endpoints return JSON responses with the following structure:

### Success Response
```json
{
  "output": "https://media.pikzels.com/..."
}
```

<Note>
  Generated images are temporary and available for approximately 24 hours. Download and host them yourself for permanent storage.
</Note>

### Error Response
```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message",
    "details": "Additional error details"
  }
}
```

## Common Error Codes

- `400` - Bad Request: Invalid or missing fields
- `401` - Unauthorized: Invalid authentication
- `402` - Payment Required: Insufficient balance
- `429` - Too Many Requests: Rate limit exceeded
- `500` - Internal Server Error
- `503` - Service Unavailable: Maintenance mode
