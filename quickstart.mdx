## Get Started with Pikzels API

This guide will walk you through making your first API call to generate an AI-powered thumbnail.

### Prerequisites

Before you begin, you'll need:
- A Pikzels API key (starts with `pkz_`)
- A tool to make HTTP requests (curl, Postman, or your favorite programming language)

<Note>
  Don't have an API key yet? Sign up at [Pikzels Dashboard](https://google.com) to get started.
</Note>

## Your First API Call

Let's generate your first AI thumbnail! We'll create a vibrant thumbnail with a simple text prompt.

<CodeGroup>

```bash cURL
curl -X POST https://api.pikzels.com/v1/thumbnail \
  -H "X-Api-Key: pkz_your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "adorable puppy playing in a sunny garden",
    "format": "16:9",
    "model": "pkz-2"
  }'
```

```python Python
import requests
import json

url = "https://api.pikzels.com/v1/thumbnail"
headers = {
    "X-Api-Key": "pkz_your_api_key_here",
    "Content-Type": "application/json"
}
data = {
    "prompt": "adorable puppy playing in a sunny garden",
    "format": "16:9",
    "model": "pkz-2"
}

response = requests.post(url, headers=headers, json=data)
result = response.json()
print(result["output"])  # URL of your generated image
```

```javascript JavaScript
const response = await fetch('https://api.pikzels.com/v1/thumbnail', {
  method: 'POST',
  headers: {
    'X-Api-Key': 'pkz_your_api_key_here',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    prompt: 'adorable puppy playing in a sunny garden',
    format: '16:9',
    model: 'pkz-2'
  })
});

const data = await response.json();
console.log(data.output); // URL of your generated image
```

```php PHP
<?php
$url = 'https://api.pikzels.com/v1/thumbnail';
$data = [
    'prompt' => 'adorable puppy playing in a sunny garden',
    'format' => '16:9',
    'model' => 'pkz-2'
];

$options = [
    'http' => [
        'header' => [
            "X-Api-Key: pkz_your_api_key_here",
            "Content-Type: application/json"
        ],
        'method' => 'POST',
        'content' => json_encode($data)
    ]
];

$context = stream_context_create($options);
$result = file_get_contents($url, false, $context);
$response = json_decode($result, true);
echo $response['output']; // URL of your generated image
?>
```

</CodeGroup>

### Response

A successful response will look like this:

```json
{
  "output": "https://media.pikzels.com/temp/pikzels_1234567890.jpeg"
}
```

The `output` field contains the URL of your generated image. This URL is temporary and should be downloaded if you need permanent storage.

<Tip>
  Generated images are available at the URL for a limited time. Download them to your server for permanent storage.
</Tip>

## Understanding the Parameters

Let's break down what each parameter does:

<ParamField body="prompt" type="string" required>
  The text description of the image you want to generate. Be specific and descriptive for best results.
</ParamField>

<ParamField body="format" type="string" required>
  The aspect ratio of your image. Options:
  - `16:9` - Landscape (great for YouTube thumbnails)
  - `9:16` - Portrait (perfect for Instagram Stories)
  - `1:1` - Square (ideal for Instagram posts)
</ParamField>

<ParamField body="model" type="string" required>
  The AI model to use. Options:
  - `pkz-1` - Legacy model: Great for gaming and vivid thumbnails (Roblox, Fortnite, GTA 5, FAR CRY, Minecraft, Pubg)
  - `pkz-2` - Go-to model: Most versatile and consistent for realistic images, photorealism, and detailed content
  - `pkz-3` - Most advanced model: Our most powerful & capable, preferred choice for highest quality results
</ParamField>

## Error Handling

Always implement proper error handling in your production code:

<CodeGroup>

```python Python
import requests

try:
    response = requests.post(url, headers=headers, json=data)
    response.raise_for_status()  # Raises an HTTPError for bad responses
    
    result = response.json()
    if "output" in result:
        print(f"Success! Image URL: {result['output']}")
    else:
        print(f"Error: {result.get('error', {}).get('message', 'Unknown error')}")
        
except requests.exceptions.HTTPError as e:
    if e.response.status_code == 401:
        print("Invalid API key. Check your authentication.")
    elif e.response.status_code == 402:
        print("Insufficient balance. Please add credits to your account.")
    elif e.response.status_code == 429:
        print("Rate limit exceeded. Please slow down your requests.")
    else:
        print(f"HTTP Error: {e}")
except Exception as e:
    print(f"An error occurred: {e}")
```

```javascript JavaScript
try {
  const response = await fetch('https://api.pikzels.com/v1/thumbnail', {
    method: 'POST',
    headers: {
      'X-Api-Key': 'pkz_your_api_key_here',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(data)
  });

  if (!response.ok) {
    const error = await response.json();
    switch(response.status) {
      case 401:
        console.error('Invalid API key');
        break;
      case 402:
        console.error('Insufficient balance');
        break;
      case 429:
        console.error('Rate limit exceeded');
        break;
      default:
        console.error('Error:', error.error?.message);
    }
    return;
  }

  const result = await response.json();
  console.log('Success! Image URL:', result.output);
  
} catch (error) {
  console.error('Request failed:', error);
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Optimize Your Prompts" icon="lightbulb">
    - Be specific and descriptive
    - Include style keywords (realistic, cartoon, minimalist)
    - Mention colors, lighting, and mood
    - Avoid contradictory descriptions
  </Accordion>
  
  <Accordion title="Handle Rate Limits" icon="gauge">
    - Maximum 10 concurrent requests per API key
    - Implement exponential backoff for retries
    - Queue requests if you need high throughput
    - Consider caching results when appropriate
  </Accordion>
  
  <Accordion title="Secure Your API Key" icon="lock">
    - Never expose your API key in client-side code
    - Use environment variables for API keys
    - Rotate keys regularly
    - Set up IP whitelisting if available
  </Accordion>
</AccordionGroup>
